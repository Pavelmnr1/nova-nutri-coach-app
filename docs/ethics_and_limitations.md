# Ethics And Limitations

## Intended Scope

This project is an educational AI nutrition assistant submission. Its main working form is a Telegram bot that provides practical, supportive food-related guidance.

## Non-Medical Boundary

- The project must not be presented as a doctor, dietitian, or diagnostic system.
- Outputs are informational and approximate.
- Users should not rely on the project for emergency, diagnostic, or treatment decisions.

## AI Transparency

- The project uses external AI APIs.
- The repository does not claim that the team trained its own large language or vision model.
- AI responses can be imperfect, incomplete, or inconsistent.

## Prototype Honesty

- The Telegram bot is the main working version in this repository.
- The web application is a web interface prototype / UI prototype / future web version concept.
- This documentation does not claim that the web prototype is fully integrated with the Telegram bot unless that is demonstrated separately.

## Data And Safety Limitations

- Food recognition and nutrition interpretation can be wrong.
- Portion estimation from text or photos is approximate.
- The quality of the result depends on user input quality and external AI output quality.
- The repository should not include real user secrets, real private conversation logs, or committed local databases.
